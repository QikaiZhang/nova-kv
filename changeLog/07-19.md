# 2026-07-19

## CRC 校验实现

### 新增
- **CRC32 校验**：`EncodeLogRecord` 写入时自动计算 CRC，`ReadLogRecord` 读取时自动验证
  - 校验范围：Type + KeySize + ValueSize + Key + Value（跳过 CRC 自身 4 字节）
  - CRC 不匹配时返回 `CRC mismatch` 错误
- **IOManager.Size()** 接口：暴露文件大小，用于读路径边界检查
- **文件边界检查**：`ReadLogRecord` 在读 header 前检查 `offset + headerReadSize` 不超出文件末尾

### 修改
- `fio/io_manager.go`：`IOManager` 接口新增 `Size() (int64, error)`
- `fio/file_io.go`：`FileIO` 实现 `Size()`，通过 `os.File.Stat()` 获取
- `data/log_records.go`：`EncodeLogRecord` 写入 CRC，`DecodeLogRecordHeader` 从 header 解析存储的 CRC
- `data/data_file.go`：`ReadLogRecord` 新增边界检查 + CRC 验证流程
- `data/log_records_test.go`：新增 CRC 编解码单元测试（4 个用例）
- `db_test.go`：`TestOpen_CRCValidation` 从 skip 改为实际测试（损坏 CRC 字节后验证检测）

### 修复
- `DecodeLogRecordHeader`：`error()` 内置调用 → `fmt.Errorf()`（修复编译错误）
- `DecodeLogRecordHeader`：`crc32.ChecksumIEEE()` 误算 CRC → `binary.LittleEndian.Uint32()` 直接读取
- `ReadLogRecord`：`return` 无值 → `return nil, 0, fmt.Errorf(...)`
- `ReadLogRecord`：`int64` 切片索引 → `int(headerSize)` 类型转换
- 清理过时代码注释（`//oldCRC`、`//补充`、`//crc 需要完整的 index`）
