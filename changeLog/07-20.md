# 2026-07-20 迭代日志

## 新增功能

### 1. Close & Sync

- `DB.Close()`：按顺序 Sync 活跃文件 → 关闭活跃文件 → 关闭所有旧文件
- `DB.Sync()`：仅 Sync 活跃文件——旧文件在写满轮转时已经持久化过，无需重复操作

### 2. 索引迭代器（Iterator）

新增 `index.Iterator` 接口，支持前缀过滤和反向遍历：

| 方法 | 说明 |
|------|------|
| `Rewind()` | 回到迭代器起点 |
| `Seek(key)` | 定位到第一个 >= key（正向）或 <= key（反向）的位置 |
| `Next()` | 移动到下一个 key |
| `Valid()` | 当前位置是否有效 |
| `Key()` | 获取当前 key |
| `Value()` | 获取当前 LogRecordPos |
| `Close()` | 释放资源 |

配置项 `IteratorOptions`：
- `Prefix`：只遍历前缀匹配的 key
- `Reverse`：是否反向遍历

BTree 实现策略：Rewind/Seek 时一次性快照收集所有匹配 item 到切片，后续遍历不持有 BTree 锁，并发安全。

### 3. ListKeys & Fold

- `ListKeys()`：从内存索引直接获取所有 key，零磁盘读取
- `Fold(fn)`：遍历全量 key-value，value 从磁盘按 LogRecordPos 读取，fn 返回 false 提前终止

### 4. WriteBatch（事务批量写入）

提供原子批量写入能力：

- `Put/Delete` 暂存到 `pendingWrites`（map）
- `Commit` 分配全局递增 SeqNo → 批量写入磁盘 → 追加 `LogRecordTxnFinished` 标记 → 更新索引
- 崩溃恢复：重启时按 SeqNo 分组暂存，仅遇到 TxnFinished 的 batch 才应用到索引

配置项 `WriteBatchOptions`：
- `MaxBatchNum`：单批次最大操作数（0=不限制）
- `SyncWrites`：Commit 时是否强制刷盘

### 5. SeqNo 编解码

`LogRecord` 新增 `SeqNo` 字段（varint 编码），磁盘格式更新为：

```
CRC(4B) | Type(1B) | KeySize(varint) | ValueSize(varint) | SeqNo(varint) | Key | Value
```

- SeqNo=0 表示普通 Put/Delete
- SeqNo>0 表示 WriteBatch 写入
- 新增 `LogRecordTxnFinished` 类型（=3）标记事务完成

`maxLogRecordHeaderSize` 从 15 调整到 25（4+1+5+5+10）。

## 修复
