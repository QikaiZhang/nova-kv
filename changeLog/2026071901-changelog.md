# Changelog

## 2026-07-19 01:00 — 修复 DecodeLogRecordHeader 的 varint 截断漏洞

### 修复内容

**文件**：`data/log_records.go`

**问题**：`DecodeLogRecordHeader` 调用 `binary.Varint` 解码 KeySize 和 ValueSize 时，未检查返回值 `n`。当文件尾部存在写入中断的残缺字节时，`binary.Varint` 返回 `(0, 0)`，导致截断的 varint 被静默当作零值接受，残缺字节被误解析为一条合法的空记录（keySize=0, valueSize=0）。在重启索引重建时，这条假记录会被插入内存索引，污染空 key 的索引条目。

**修复**：
1. 在两个 `binary.Varint` 调用后各增加 `if n <= 0 { return nil, 0 }` 守卫
2. 新增导出错误 `ErrIncompleteHeader`，便于调用方区分散"文件尾部自然空隙"和"真正的数据损坏"

**影响范围**：
- `DecodeLogRecordHeader`：截断 varint 不再返回假 header，改为返回 nil
- `ReadLogRecord`：已处理 header 解码失败的路径，无需修改
- `loadIndexFromDataFiles`：已有 corruptedRecords 收集逻辑，尾部空隙被正确识别为损坏记录而非有效数据
